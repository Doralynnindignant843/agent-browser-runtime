## 🧠 Managing agent browser sessions

The software organizes browser activity into groups. This keeps the agent work separate from your personal browsing.

### Create a new browser session
When you open an agent, the runtime will prompt you to create a new session. Set a name for your agent. This helps you track which agent uses which tab group. The runtime assigns a specific browser profile to this agent. This keeps cookies and history secure for that specific automation task.

### Monitor agent progress
You can see exactly what the agent sees. The software provides a window into the live browser tab. You can pause the agent at any time by clicking the "Suspend" button. If the agent finishes a task, the status light will turn green.

### Use human handoff
If the agent encounters a login screen, a Captcha, or a request for two-factor authentication, it will stop automatically. You can then click the "Take Control" button. This allows you to log in manually. Once you finish the login, click "Resume" to let the agent continue its work.

## ❓ Frequently asked questions

### Does this software use my personal Chrome history?
No. The runtime creates a separate, containerized profile. Your personal history, passwords, and bookmarks remain private and separate from the agent activity.

### Can I run multiple agents at once?
Yes. You can manage multiple tab groups simultaneously. Each group operates independently. Each group has its own memory and storage to prevent interference between tasks.

### What should I do if the software crashes?
Close the application completely and restart it. If the issue persists, check that your Google Chrome is up to date. Occasionally, browser updates change how the software interacts with the sessions.

### Where can I find more help?
The software includes a basic log file. If you encounter errors, you can view this file by navigating to the "Settings" menu inside the application. This text file provides details about what happened when the crash occurred.

## 📁 File structure reference

You do not need to edit these files, but they serve specific purposes within the installation directory:

- runtime.exe: The core engine that manages the browser sessions.
- profiles/: A folder that holds the individual data for each created agent.
- logs/: This folder stores text files regarding application performance.
- config.json: A simple file that stores your preferences.
